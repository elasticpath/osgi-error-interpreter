This error means that bundle declares a reference to a but there is no active bundle available that provides this service. This can also mean that there is a circular dependency between bundles that require services from each other.

To solve this error, ensure that the bundle that provides a service for has been deployed into the OSGi container and is in the active state. If the bundles depend on services from each other, declare one of the service references as availability="optional" so that it becomes late-bound.
